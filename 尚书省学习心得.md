# 尚书省学习心得：claw-code 架构设计

## 核心感悟

### 1. 工具编排的"三层架构"

claw-code 的工具编排采用清晰的三层设计：

```
┌─────────────────────────────────────────┐
│  Python 层：PortRuntime + QueryEngine   │  ← 路由和查询
├─────────────────────────────────────────┤
│  Rust 层：ConversationRuntime           │  ← 核心循环
├─────────────────────────────────────────┤
│  插件层：PluginRegistry + HookRunner    │  ← 扩展机制
└─────────────────────────────────────────┘
```

**启示**：我们在设计多 Agent 调度时，也应该考虑这种分层解耦。路由层负责"理解意图"，核心层负责"执行循环"，扩展层负责"定制行为"。

### 2. Trait 抽象的力量

Rust 端的 `ApiClient` 和 `ToolExecutor` 都是 trait：

```rust
pub trait ApiClient {
    fn stream(&mut self, request: ApiRequest) -> Result<Vec<AssistantEvent>, RuntimeError>;
}

pub trait ToolExecutor {
    fn execute(&mut self, tool_name: &str, input: &str) -> Result<String, ToolError>;
}
```

**启示**：这种设计使得：
- 单元测试时可以注入 Mock 实现
- 生产环境可以切换不同的 API 提供商
- 工具执行可以支持本地、远程、沙箱等多种模式

### 3. Hook 系统的巧妙设计

PreToolUse 和 PostToolUse 两级 Hook：

```rust
pub fn run_pre_tool_use(&self, tool_name: &str, tool_input: &str) -> HookRunResult
pub fn run_post_tool_use(&self, tool_name: &str, tool_input: &str, tool_output: &str, is_error: bool) -> HookRunResult
```

Hook 命令返回：
- Exit 0 = Allow
- Exit 2 = Deny
- Exit 其他 = Warn（继续执行）

**启示**：这种"退出码语义"设计非常巧妙，任何语言都可以实现 hook，无需引入复杂 SDK。

### 4. 会话管理的版本化思维

```rust
pub struct Session {
    pub version: u32,           // 版本号
    pub messages: Vec<ConversationMessage>,
}
```

**启示**：一开始就考虑版本化，为未来的格式迁移留下空间。这点很容易被忽视，但对长期维护很重要。

### 5. 插件系统的"聚合"模式

```rust
impl PluginRegistry {
    pub fn aggregated_hooks(&self) -> Result<PluginHooks, PluginError>
    pub fn aggregated_tools(&self) -> Result<Vec<PluginTool>, PluginError>
}
```

**启示**：每个插件只声明自己的部分，Registry 负责聚合。这种"声明式聚合"比"集中配置"更灵活。

## 对我们 OpenClaw 的借鉴

### 可以借鉴的地方

1. **ToolExecutor Trait**：为 OpenClaw 的工具执行层引入 trait，支持本地/远程/沙箱执行
2. **Hook 系统**：实现 PreToolUse/PostToolUse 两级 Hook
3. **Session 版本化**：为会话数据结构添加 version 字段
4. **上下文压缩**：实现 `compact_session()` 防止上下文溢出

### 需要注意的地方

1. Python 端的"快照占位符"模式不够优雅，真正的逻辑在 TypeScript Archive 中
2. 插件系统缺乏真正的沙箱隔离
3. 多会话服务器使用简单的 HashMap，生产环境需要分布式存储
